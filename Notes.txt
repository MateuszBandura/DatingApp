json webtoken industry standard for tokens sugested in some RFC 7519
they are selfcontained and they contain credentials, claims and others
they are basically a long string seperated in to 3 parts with dots
first part is the header of the token, it contains algorithm and type of token(JWT)(the algorithm is used to encrypt the signature in the third part of the token)
second part is the payload, this the place where we contain our claims and credentials
we also have 3 time stamps in there, nbf sets the date and time before which token can not be used,
exp sets the expiration date, iat sets when the token was issued at 
third part conatins encrypted signature (its encrypted by the server with a secure key that never ever leaves the server)

user sends credentials (login+password) -> server validates the credentials and returns JWT -> user then sends JWT with every single request to prove that hes authenticated -> server checks the JWT and sends back the response

scoped lifetime is the one that is usually used for JWT creation service
symmetric key is a key that is used both for encrypting and decrypting information
